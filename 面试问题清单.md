# 校园预约系统面试问题清单

## 一、项目整体架构与设计

### 1.1 项目背景与需求
- 你为什么选择开发这个校园预约系统?主要解决了什么痛点?
- 这个项目的主要用户群体是谁?他们有哪些核心需求?
- 项目的业务流程是怎样的?能简单描述一下吗?

### 1.2 架构设计
- 你采用了前后端分离架构,能说说这种架构的优势吗?
- 为什么选择Spring Boot 3.5.10而不是其他框架?
- 项目的整体架构是怎样的?能画个架构图吗?
- 如果要扩展成微服务架构,你会如何拆分服务?

### 1.3 技术选型
- 为什么选择MyBatis而不是JPA或Hibernate?
- 为什么使用Redis而不是其他缓存方案?
- JWT相比传统的Session认证有什么优势?
- 为什么选择Swagger/Knife4j而不是其他API文档工具?

## 二、用户认证与安全

### 2.1 JWT认证

### 2.3 密码安全

### 2.4 其他安全问题

## 面试准备建议

### 1. 核心重点
- **必须准备**: JWT认证、邮箱验证码、Redis缓存、MyBatis、数据库设计
- **重点准备**: 并发处理、性能优化、安全防护、异常处理
- **了解即可**: 前端技术、部署运维、项目管理

### 2. 回答技巧
- 使用STAR法则(Situation-Task-Action-Result)回答问题
- 举具体的例子,不要泛泛而谈
- 展示思考过程,而不仅仅是结果
- 诚实回答,不知道就说不知道,但要展示学习意愿

### 3. 项目展示
- 准备项目架构图
- 准备核心流程图
- 准备数据库ER图
- 准备关键代码片段
- 准备性能测试数据

### 4. 反问环节
- 准备一些有深度的问题,如:
  - 团队的技术栈是什么?
  - 团队的开发流程是怎样的?
  - 团队面临的最大技术挑战是什么?
  - 公司对新人的培养计划是什么?

## 总结

这份面试问题清单涵盖了项目相关的各个方面,从基础的技术细节到深度的系统设计,从具体的实现方案到开放性的思考题。建议你:

1. **深入理解**: 对自己负责的模块要非常熟悉
2. **准备案例**: 准备具体的案例和数据支撑
3. **思考深度**: 不仅要知道怎么做,还要知道为什么这么做
4. **展示亮点**: 突出自己的技术亮点和创新点
5. **诚实谦虚**: 不懂的地方要诚实,但要展示学习能力

祝你面试顺利!
